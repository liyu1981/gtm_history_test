This page shows how to setup GTM with history change type tag for single page app.

Single page app trends to use HTML5's history API to change the URL when its content changes, which makes setup GTM difficulty.

Take Facebook's standard event 'ViewContent', if we only want to fire it after the URL changes to specific pattern, such as with `step=3` in URL in our example, we need some advanced setting in GTM.

The steps are:

  • Create a GTM trigger with history change type, and setup it to fire when PAGE URL contains `step=3`.
  • Create a tag in GTM with trigger just created, and setup its custom HTML as follows
    •
